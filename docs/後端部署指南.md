# Brainwave 後端部署說明（3 種情境）

## 1) 本地部署 + 本地資料庫

### Step 1: 準備本地 MySQL
- 建立資料庫（例如 `brainwave`）
- 確認可用帳密（預設示例：`root/root`）

### Step 2: 啟動後端（local profile）
- 直接在 IDE 啟動 `BrainwaveBackendApplication`
- 或命令列執行 `mvn spring-boot:run`
- `application.properties` 預設會走 `local` profile

### Step 3: 本地驗證
- 健康檢查：`http://localhost:8080/api/health`
- 確認 `users`、`members`、`flyway_schema_history` 等表由 Flyway 建立

---

## 2) Zeabur 部署 + Zeabur 資料庫

### Step 1: 建立 Zeabur 服務
- Zeabur -> Add Service -> 選 GitHub 後端 repo
- Root Directory 使用 repo 根目錄

### Step 2: 設定 Zeabur 環境變數（必填）
- `SPRING_PROFILES_ACTIVE=prod`
- `DB_URL=jdbc:mysql://<host>:<port>/<db>?useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Taipei`
- `DB_USERNAME=<username>`
- `DB_PASSWORD=<password>`
- `APP_CORS_ALLOWED_ORIGINS=https://your-frontend-domain.vercel.app`

若你也要本機前端測線上後端，可改為：
- `APP_CORS_ALLOWED_ORIGINS=http://localhost:3000,https://your-frontend-domain.vercel.app`

### Step 3: 部署後驗證
- 健康檢查：`https://<your-backend>.zeabur.app/api/health`
- 確認資料表由 Flyway 建立成功

---

## 3) 本地部署 + Zeabur 資料庫

### 使用情境
- 本機 debug 程式，但要連線到 Zeabur MySQL 測試真實資料

### 建議做法（擇一）
- 在 IDE/系統環境變數臨時設定：
  - `SPRING_PROFILES_ACTIVE=prod`
  - `DB_URL`、`DB_USERNAME`、`DB_PASSWORD`（填 Zeabur DB）
  - `APP_CORS_ALLOWED_ORIGINS=http://localhost:3000`
- 或在啟動參數加上：
  - `--spring.profiles.active=prod`

### 注意事項
- 這種模式會直接動到線上資料庫，請避免執行大量測試寫入
- 優先使用測試用 Zeabur DB，不要連正式資料庫
